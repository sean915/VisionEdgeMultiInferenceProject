IVS (Intelligent Vision Sensor) 구조 설명서 (based on LGESHM PKG Prewelding)

1. 시스템 개요

HM에서 수행된 PKG공정 약용접/소착 검출을 위해 개발된 IVS 기반으로 sw 구조 성명 자료입니다.

3. 데이터 흐름 (Data Flow)

%%{init: {'theme':'base', 'themeVariables': {'fontSize':'12px'}, 'sequence':{'actorMargin':40, 'boxMargin':5, 'width':120}}}%%
sequenceDiagram
    participant CAM as Camera
    participant FT as Frame Task
    participant TT as Trigger Task
    participant PLC as PLC Task
    participant AT as Abnormal Task
    participant IO as IO Task

    CAM->>FT: Video Stream
    loop Every Frame
        FT->>TT: Frame (np.ndarray)
        TT->>TT: Detect Object & Track
        alt Object Detected
            TT->>PLC: Request Cell ID
            PLC-->>TT: Cell ID + Pallete
            TT->>AT: Trigger Frame + Metadata
            AT->>AT: Defect Detection
            alt Defect Found
                AT->>PLC: Send Alarm
                AT->>IO: Save Detection
                AT->>IO: Write CSV/Log
            else No Defect
                AT->>IO: Save Normal
                AT->>IO: Write CSV/Log
            end
            AT->>IO: Video Frame Buffer
        end
    end
    
    loop Every 2 seconds
        PLC->>PLC: Read Busy Status
        PLC->>PLC: Send Heartbeat (0↔1)
    end

5. Task 별 상세 명세

5.1 Frame Task

핵심 역할: 카메라/RTSP 스트림에서 프레임을 읽어 Frame Queue에 공급하는 프레임 생산자(Producer) 역할. 스트림 연결 실패 시 자동 재연결을 시도하며, 큐가 가득 차면 새 프레임을 버립니다 (drop).

Frame Task Input/Output

항목 타입 설명 크기/형식
INPUT
Camera Source str 카메라 경로 또는 RTSP URL rtsp://192.168.0.100:554/stream
자동 취득 (스트림에서 읽음)
FPS float 스트림에서 자동으로 읽어온 프레임레이트 stream.fps (예: 30.0)
Resolution tuple 스트림에서 자동으로 읽어온 해상도 (width, height) (예: 1920x1080)
OUTPUT
Frame np.ndarray RGB 이미지 프레임 shape=(H, W, 3), dtype=uint8
Timestamp float 프레임 타임스탬프 time.time()
Frame Queue Queue 프레임 버퍼 maxsize=30

Note: FPS와 Resolution은 입력 파라미터가 아닙니다. 카메라 소스(RTSP URL 또는 동영상 파일)에 연결 후, stream.fps로 자동으로 읽어옵니다.

5.2 Trigger Task

핵심 역할: 프레임에서 용접 대상 객체를 검출하고 추적하여 캡처 시점을 결정하는 트리거 생성자 역할. YOLO 모델로 객체를 검출하고, TriggerTracker로 상태를 추적하여 적절한 시점에 Abnormal Task로 전달합니다. capture_flag 또는 capture_end_flag가 True일 때만 Trigger Queue에 데이터를 전송합니다.

Trigger Task Input/Output

항목 타입 설명 형식
INPUT
Frame Queue Queue[np.ndarray] 입력 프레임 스트림 shape=(H,W,3)
Trigger Model str ONNX 모델 경로 trigger_model.onnx
Reference Image str 카메라 기준 이미지 reference.jpg
ROI Settings list[ShapeDTO] 관심 영역 설정 [{"type":"box","data":[[x1,y1],[x2,y2]]}]
Cell ID Queue Queue[str] PLC로부터 받은 Cell ID maxsize=1
Pallete Queue Queue[int] PLC로부터 받은 Pallete 번호 maxsize=1
PROCESSING
Detection DetectionResultDTO 객체 검출 결과 boxes, labels, scores, masks
Tracking TriggerTracker 객체 추적 상태 track_id, bbox, state
Shift Detection bool 카메라 이동 감지 SSIM < threshold
Capture Decision bool 캡처 신호 생성 based on tracking logic
OUTPUT
Trigger Queue Queue[TriggerDTO] 트리거 정보 maxsize=10

TriggerDTO 구조:

class TriggerDTO:
    capture_flag: bool              # 캡처 중 신호 (True: 검출 수행)
    capture_end_flag: bool          # 캡처 종료 신호 (True: 결과 집계 및 저장)
    cell_id: str                    # Cell ID (15자)
    frame: np.ndarray               # 시각화 프레임 (마킹됨)
    frame0: np.ndarray              # 원본 프레임
    is_shift: bool                  # 카메라 이동 여부
    shift_log: dict                 # 이동 상세 정보
    elapsed_time: float             # 처리 시간

Trigger Queue 전송 조건 (실제 코드 기준):

  • capture_flag=True: 객체 추적 중, Abnormal Task에서 결함 검출 수행
  • capture_end_flag=True: 추적 종료, Abnormal Task에서 결과 집계 및 파일 저장
  • 둘 다 False면 Trigger Queue에 전송하지 않음 (Display만 업데이트)

5.3 Abnormal Task

핵심 역할: 트리거된 프레임에서 용접 결함을 검출하고, 결과를 PLC 알람으로 전송 및 파일로 저장하는 결함 판정자 역할. capture_flag=True 프레임에서 결함 검출을 수행하고, capture_end_flag=True일 때 결과를 집계하여 저장합니다.

Abnormal Task Input/Output

항목 타입 설명 형식
INPUT
Trigger Queue Queue[TriggerDTO] 트리거 프레임 maxsize=10
Abnormal Model str ONNX 결함 검출 모델 abnormal_model.onnx
Score Thresholds list[float] 결함별 점수 임계값 [0.3, 0.3, 0.3] (약용접, 소착1, 소착2)
Size Thresholds list[int] 결함별 크기 임계값 (px²) [500, 500, 500]
PROCESSING
Detection DetectionResultDTO 결함 검출 결과 boxes, labels, scores, masks
Filtering list[bool] 임계값 통과 여부 score >= threshold AND size >= threshold
Tracking AbnormalTracker 결함 추적 track_id, defect_type, state
Alarm Calculation int 알람 코드 계산 결함 타입별 매핑
OUTPUT
Control Queue Queue[int] PLC 알람 값 0(정상) or 1~N(불량)
Event Queue Queue[dict] 저장 이벤트 {"type":"detect", "data":...}
Marked Image np.ndarray 결함 마킹 이미지 bbox + mask overlay
Detection Log .log 파일 텍스트 로그 한 줄씩 추가
Detection CSV .csv 파일 구조화된 데이터 CSV 형식

DetectionResultDTO 구조:

class DetectionResultDTO:
    boxes: np.ndarray      # [N, 4] (x1, y1, x2, y2)
    labels: np.ndarray     # [N] 클래스 인덱스
    scores: np.ndarray     # [N] 신뢰도 점수
    masks: np.ndarray      # [N, H, W] 세그멘테이션 마스크

알람 값 매핑:

결함 타입 Label Alarm Value
정상 (OK) - 0
불량 (NG) 0/1/2 (약용접/소착1/소착2) 1

Note: 실제 PLC로 전송되는 알람 값은 0 (정상) 또는 1 (불량) 만 사용합니다.
결함 타입(약용접, 소착1, 소착2)은 내부 로그 및 이미지 저장용으로만 구분됩니다.

5.4 PLC Task

핵심 역할: PLC와의 양방향 통신을 담당하는 태스크.

  • 읽기: Cell ID, Pallete 번호, Busy 상태를 주기적으로 읽어 다른 Task에 제공
  • 쓰기: Abnormal Task로부터 받은 Alarm 값을 PLC에 전송, Heartbeat로 연결 상태 유지
  • 통신 장애 시 자동 재연결을 시도하며, 모든 통신 이력을 로그 파일로 기록합니다.

Heartbeat와 Machine Status 구분:

  • Heartbeat (쓰기): IVS → PLC로 생존 신호 전송 (0↔1 토글, 2초 주기)
  • Machine Status (읽기): PLC → IVS로 설비 동작 상태 확인 (Busy 값 읽기, 2초 주기)

작업 주기 목적

Machine Status 읽기 2000ms 설비 동작 상태 확인
Alarm 송신 Event-driven 결함 검출 즉시 전송

5.5 IO Task

핵심 역할: 비디오 저장 및 스토리지 용량 관리를 담당하는 태스크.

  • 트리거 전후 10초 분량의 프레임을 버퍼링하여 MP4 비디오로 인코딩/저장
  • 디스크 용량을 모니터링하여 임계치 초과 시 오래된 폴더부터 자동 삭제 (FIFO)

⚠️ 주의: 이미지 저장 및 CSV/Log 기록은 Abnormal Task에서 직접 처리합니다.

항목 타입 설명 형식
INPUT
Event Queue Queue[StreamEventDTO] 프레임 + 메타데이터 Abnormal/Trigger Task에서 송신
Stream Buffer deque[np.ndarray] 프레임 누적 버퍼 최근 10초 프레임

FPS int 비디오 프레임레이트 25 FPS
PROCESSING
Video Encoding cv2.VideoWriter MP4 인코딩 codec=mp4v, fps=25
Disk Monitoring psutil.disk_usage() 디스크 사용률 체크 60초마다
OUTPUT
Video Clip .mp4 트리거 전후 영상 video_YYMMDD_HHMMSS.mp4
Dashboard Event DetectHistoryVideoDTO UI 업데이트 (선택) Dashboard Queue로 전송
STORAGE MANAGEMENT
Disk Usage Check 주기적 디스크 사용률 모니터링 60초마다
Auto Cleanup FIFO 용량 초과 시 자동 삭제 가장 오래된 폴더부터

⚠️ 중요: IO Task는 비디오만 저장합니다.

다른 로그들은 Abnormal Task에서 직접 처리:

  • 이미지 저장 (원본 + 마킹)
  • CSV 파일 기록 (검출 결과 구조화 데이터)
  • Log 파일 기록 (검출 이력 텍스트 로그)
  • 판정 결과 기록 (정상/의심/불량)

DetectHistoryDTO 구조:

class DetectHistoryDTO:
    time: datetime                   # 검출 시각
    cell_id: str                     # Cell ID
    path: str | None                 # 마킹 이미지 경로
    org_path: str | None             # 원본 이미지 경로
    mask: str | None                 # RLE 인코딩된 마스크
    detects: list[DetectInfoDTO]     # 검출 상세

class DetectInfoDTO:
    defect_type: int                 # 0:정상, 1:불량
    label: int                       # 결함 클래스 인덱스
    label_name: str                  # 약용접/소착1/소착2
    score: float                     # 신뢰도 점수
